二十三种设计模式

二十三种
发布日期:   2021-11-22

一、单例模式

​ 一个单一的类，该类负责创建自己的对象，同时确保只有单个对象被创建。这个类提供了一种访问其唯一的对象的方式，可以直接访问，不需要实例化该类的对象。

1、饿汉模式

加载到内存后，就实例化一个单例，JVM线程安全

//静态变量初始化
public class Singleton {
    
    private  static  final  Singleton INSTANCE = new Singleton();
    
    private Singleton(){}
    
    public static Singleton getInstance(){ return INSTANCE ;}
    
}

或者是

//静态代码块初始化
public class Singleton {
    
    private  static  final  Singleton INSTANCE;
    
    static {
        INSTANCE = new Singleton();    
    }
    
    private Singleton(){}
    
    public static Singleton getInstance(){ return INSTANCE ;}
    
}

2、懒汉模式

使用时才初始化（但是带来线程安全问题）

//双重判断
public class Singleton {

    private static volatile Singleton INSTANCE;

    private Singleton(){}

    public static Singleton getInstance(){
        if(INSTANCE == null){
            synchronized (Singleton.class){
                if(INSTANCE == null){
                    INSTANCE = new Singleton();
                }
            }
        }
        return INSTANCE;
    }

}

或者是

//静态内部类
public class Singleton {
    
    private Singleton(){}
    
    private static class SingletonHolder{
        private final static Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance(){
        return SingletonHolder.INSTANCE;
    }

}

3、枚举单例

枚举类没有构造方法，可以防止反序列化

//枚举单例模式
public enum Singleton {

    INSTANCE;

    public void doSomething() {
        System.out.println("doSomething");
    }

}

二、策略模式

​ 在策略模式（Strategy Pattern）中，一个类的行为或其算法可以在运行时更改。这种类型的设计模式属于行为型模式。

​ 在策略模式中，我们创建表示各种策略的对象和一个行为随着策略对象改变而改变的 context 对象。策略对象改变 context 对象的执行算法。

​ 简单来说，就是定义一个接口。实现类根据需求，实现接口方法

public interface Strategy {
   public int doOperation(int num1, int num2);
}
public class OperationAdd implements Strategy{
   @Override
   public int doOperation(int num1, int num2) {
      return num1 + num2;
   }
}
public class OperationSubtract implements Strategy{
   @Override
   public int doOperation(int num1, int num2) {
      return num1 - num2;
   }
}
public class OperationMultiply implements Strategy{
   @Override
   public int doOperation(int num1, int num2) {
      return num1 * num2;
   }
}
public class Context {
   private Strategy strategy;
 
   public Context(Strategy strategy){
      this.strategy = strategy;
   }
 
   public int executeStrategy(int num1, int num2){
      return strategy.doOperation(num1, num2);
   }
}
public class StrategyPatternDemo {
   public static void main(String[] args) {
      Context context = new Context(new OperationAdd());    
      System.out.println("10 + 5 = " + context.executeStrategy(10, 5));
 
      context = new Context(new OperationSubtract());      
      System.out.println("10 - 5 = " + context.executeStrategy(10, 5));
 
      context = new Context(new OperationMultiply());    
      System.out.println("10 * 5 = " + context.executeStrategy(10, 5));
   }
}

三、工厂模式

核心本质：

​ 实例化对象不使用new，用工厂方法代替；将选择实现类，创建统一管理和控制。从而将调用者跟我们的实现类解耦。

1、简单工厂模式

​ 设计出产品模型（接口），根据模型设计不同产品（实现类），工厂拿到产品订单（实现类名）生产对应产品（实现类对象）。

​ 确点，添加产品，不仅需要添加新的产品，还要改动工厂类（违反了“开闭原则”）。

public interface Car {
   void name();
}
public class AstonMartin implements Car {
   @Override
   public void name() {
      System.out.println("007 专属座驾！");
   }
}
public class Mitsubishi implements Car {
   @Override
   public void name() {
      System.out.println("成龙 御用！");
   }
}
public class CarFactory {
    
   //使用 getShape 方法获取形状类型的对象
   public static Car getCar(String carName){
      if(carName == null){
         return null;
      }        
      if(carName.equalsIgnoreCase("AstonMartin")){
         return new AstonMartin();
      } else if(carName.equalsIgnoreCase("Mitsubishi")){
         return new Mitsubishi();
      } 
      return null;
   }
}
public class FactoryDemo {
 
   public static void main(String[] args) {
      Car astonMartin = CarFactory.getCar("AstonMartin");
 	  astonMartin.name();
   }
   
}

2、工厂方法模式

在简单方法模式中，把工厂抽象了出来。

缺点：不仅多了产品，还多了工厂

public interface CarFactory{
    Car getCar();
}
public class AstonMartinFactory implements CarFactory {
   @Override
   public void getCar() {
      return new AstonMartin();
   }
}
public class FactoryDemo {
 
   public static void main(String[] args) {
      Car astonMartin = new AstonMartinFactory();
 	  astonMartin.name();
   }
   
}

3、抽象工厂模式

定义一个超级工厂接口，接口中有一些列的相关产品工厂。

例如：护肤品工厂，会生成面膜、眼霜、面霜、生体乳等系列产品，需要不同的工厂生产。

public interface SkinCareProductsFactory{
   
    MaskFactory getMaskFactory();
    
    EyeCreamFactory getEyeCreamFactory();
    
    BodyLotionsFactory getBodyLotionsFactory();
    
}
public class Demo {
 
   public static void main(String[] args) {
      SkinCareProductsFactory Lancome = new LancomeSkinCareProductsFactory();
 	  MaskFactory lancomeMaskFactory = Lancome.getMaskFactory();
      Mask lancomeMask = lancomeMaskFactory.getMaskProduct();
       
      EyeCreamFactory lancomeEyeCreamFactory = Lancome.getEyeCreamFactory();
      EyeCream lancomeEyeCream = lancomeMaskFactory.getEyeCreamProduct();
   }
   
}

四、建造者模式（Builder Pattern）

建造者模式，顾名思义，建造者会根据建造需要，通过不同的材料，组合子一起，就可以完成本次建造。

如装修公司，可以根据用户需求去装修房屋。虽装修风格的不同，但是装修的东西都差不多。如衣柜、床、沙发、桌椅等。简单的来说，就是把需要的家具组装起来，只是把已有的物件，更加所需选择不同风格的物品。如中式风格，沙发可能就会选择中式风格的红木制的沙发，加上一些中式纹饰的坐垫靠背等。当然如果是对中式风格极为喜爱的雅士，怎么能少了中式的茶几和极具文人雅士的书桌。欧式风格就按欧式的风格选，如维多利亚风格。也可以选择朋克风。

//Item接口
public interface Item {
   public String name();
   public Packing packing();
   public float price();    
}
//打包接口
public interface Packing {
   public String pack();
}
//纸质包装
public class Wrapper implements Packing {
 
   @Override
   public String pack() {
      return "Wrapper";
   }
}
//瓶装
public class Bottle implements Packing {
 
   @Override
   public String pack() {
      return "Bottle";
   }
}
//汉堡抽象类
public abstract class Burger implements Item {
 
   @Override
   public Packing packing() {
      return new Wrapper();
   }
 
   @Override
   public abstract float price();
}
//冷饮抽象类
public abstract class ColdDrink implements Item {
 
    @Override
    public Packing packing() {
       return new Bottle();
    }
 
    @Override
    public abstract float price();
}
//素汉堡
public class VegBurger extends Burger {
 
   @Override
   public float price() {
      return 25.0f;
   }
 
   @Override
   public String name() {
      return "Veg Burger";
   }
}
//鸡肉汉堡
public class ChickenBurger extends Burger {
 
   @Override
   public float price() {
      return 50.5f;
   }
 
   @Override
   public String name() {
      return "Chicken Burger";
   }
}
//可口可乐
public class Coke extends ColdDrink {
 
   @Override
   public float price() {
      return 30.0f;
   }
 
   @Override
   public String name() {
      return "Coke";
   }
}
//百事可乐
public class Pepsi extends ColdDrink {
 
   @Override
   public float price() {
      return 35.0f;
   }
 
   @Override
   public String name() {
      return "Pepsi";
   }
}
//餐食
public class Meal {
   private List<Item> items = new ArrayList<Item>();    
 
   public void addItem(Item item){
      items.add(item);
   }
 
   public float getCost(){
      float cost = 0.0f;
      for (Item item : items) {
         cost += item.price();
      }        
      return cost;
   }
 
   public void showItems(){
      for (Item item : items) {
         System.out.print("Item : "+item.name());
         System.out.print(", Packing : "+item.packing().pack());
         System.out.println(", Price : "+item.price());
      }        
   }    
}
public class MealBuilder {
 
   public Meal prepareVegMeal (){
      Meal meal = new Meal();
      meal.addItem(new VegBurger());
      meal.addItem(new Coke());
      return meal;
   }   
 
   public Meal prepareNonVegMeal (){
      Meal meal = new Meal();
      meal.addItem(new ChickenBurger());
      meal.addItem(new Pepsi());
      return meal;
   }
}
public class BuilderPatternDemo {
   public static void main(String[] args) {
      MealBuilder mealBuilder = new MealBuilder();
 
      Meal vegMeal = mealBuilder.prepareVegMeal();
      System.out.println("Veg Meal");
      vegMeal.showItems();
      System.out.println("Total Cost: " +vegMeal.getCost());
 
      Meal nonVegMeal = mealBuilder.prepareNonVegMeal();
      System.out.println("\n\nNon-Veg Meal");
      nonVegMeal.showItems();
      System.out.println("Total Cost: " +nonVegMeal.getCost());
   }
}

五、原型模式

简单地说就是复制粘贴，就像经过反复修改的配方，最终达到了理想的配比，以后就可以直接根据配方复制出同样的产品就是了，不用再花时间，重投来过。

//克隆实现Cloneable接口
public abstract class Shape implements Cloneable {
   
   private String id;
   protected String type;
   
   abstract void draw();
   
   public String getType(){return type;}
   
   public String getId() {return id;}
   
   public void setId(String id) { this.id = id;}
   
   public Object clone() {
      Object clone = null;
      try {
         clone = super.clone();
      } catch (CloneNotSupportedException e) {
         e.printStackTrace();
      }
      return clone;
   }
}

六、适配器模式

可以理解为一种转换器，如国内电器是220V电压，而国外的电压不能满足，既不能直接插电使用，中间需要添加一个电压转换器，这个电压转换器，就是适配器模式

//适配类 网线
public class Adaptee{
    public void request(){
        system.out.println("连接上了网线!");
    }
}
//USB转换器接口
public interface NetToUsb{
    
    public void handleRequest();
    
}
//适配器，连上网线
public class Adapter implements NetToUsb{
    private Adaptee adaptee;
    
    public Adapter(Adaptee adaptee){
        this.adaptee = adaptee;
    }
    
    public void handleRequest(){
        super.request();
    }
}
public class Computer{
    public void net(NetTOUsb adapter){
        adapter.handleRequest();
    }
}
public class Demo{
	
    public static void main(String[] args) {
        Adapter adapter = new Adapter(new Adaptee());
        Computer computer = new Computer();
        computer.net(adapter);
    }
    
}

七、装饰器模式

简单来说就是，不改变原有对象，但可以增添其他功能（也就是说，对修改关闭，对扩展开放）

public interface Shape {
   void draw();
}
public class Circle implements Shape {
 
   @Override
   public void draw() {
      System.out.println("Shape: Circle");
   }
}
//对Shap类的装饰类，抽象出来
public abstract class ShapeDecorator implements Shape {
   protected Shape decoratedShape;
 
   public ShapeDecorator(Shape decoratedShape){
      this.decoratedShape = decoratedShape;
   }
 
   public void draw(){
      decoratedShape.draw();
   }  
}
public class RedShapeDecorator extends ShapeDecorator {
 
   public RedShapeDecorator(Shape decoratedShape) {
      super(decoratedShape);     
   }
 
   @Override
   public void draw() {
      decoratedShape.draw();         
      setRedBorder(decoratedShape);
   }
 
   private void setRedBorder(Shape decoratedShape){
      System.out.println("Border Color: Red");
   }
}

八、代理模式（Proxy Pattern）

代理模式，就是相当于委托专业人士，帮你办理，你要去办的事。如，买房子找中介，你并不需要直接找开发商，房屋贷款也是中介帮你办理。即达到相同的目的，有减少了一些复杂的处理，又可以根据需要，添加特殊的处理。

代理模式又分为：静态代理、动态代理

1、静态代理

public interface RentHouse {
    
    void rentHouse();
    
}
public class Landlord implements RentHouse{
    @Override
    public void rentHouse() {
        System.out.println("房东要出租房子！");
    }
}
public class Proxy implements RentHouse{
    
    private RentHouse landlord;
    
    private Proxy(Landlord landlord){
        this.landlord = landlord;
    }
    
    @Override
    public void rentHouse() {
        lookHouse();
        landlord.rentHouse();
    }
    
    private void lookHouse(){
        System.out.println("带人看房");
    }
    
}

2、动态代理

主要分为两类：基于接口的动态代理（JDK动态代理），基于类的动态代理（ cglib ），JAVAssist（JBoss）

public class ProxyInvocationHandler implements InvocationHandler {

    private RentHouse target;

    public void setTarget(RentHouse target){
        this.target = target;
    }

    public Object getProxy(){
        return Proxy.newProxyInstance(this.getClass().getClassLoader(), target.getClass().getInterfaces(),this);
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("带租客看房。");
        Object result = method.invoke(target , args);
        return result;
    }
}
@Test
public void demo() {
    Landlord landlord = new Landlord();
    ProxyInvocationHandler proxy = new ProxyInvocationHandler();
    proxy.setTarget(landlord);
    RentHouse rentHouse =(RentHouse) proxy.getProxy();
    rentHouse.rentHouse();
}

九、外观模式

又名门面模式，是一种通过为多个复杂的子系统提供一个一致的接口，而使这些子系统更加容易被访问的模式。该模式对外有一个统一的接口，隐藏了内部子系统的具体细节，降低的复杂度，提高了可维护性。（简单来说就是，买股票和买基金的原理。买股票需要了解很多的信息和一些专业的金融知识，还要明确知道买卖的操作。买基金就是这个模式的体现）

十、桥接模式

桥接（Bridge）是用于把抽象化与实现化解耦，使得二者可以独立变化。这种类型的设计模式属于结构型模式，它通过提供抽象化和实现化之间的桥接结构，来实现二者的解耦。

这种模式涉及到一个作为桥接的接口，使得实体类的功能独立于接口实现类。这两种类型的类可被结构化改变而互不影响。

//1、创建桥接实现接口
public interface DrawAPI {
   public void drawCircle(int radius, int x, int y);
}

//2、创建实现了 DrawAPI 接口的实体桥接实现类。
public class RedCircle implements DrawAPI {
   @Override
   public void drawCircle(int radius, int x, int y) {
      System.out.println("Drawing Circle[ color: red, radius: "
         + radius +", x: " +x+", "+ y +"]");
   }
}
//3、使用 DrawAPI 接口创建抽象类 Shape。
public abstract class Shape {
   protected DrawAPI drawAPI;
   protected Shape(DrawAPI drawAPI){
      this.drawAPI = drawAPI;
   }
   public abstract void draw();  
}
//4、创建实现了 Shape 抽象类的实体类。
public class Circle extends Shape {
   private int x, y, radius;
 
   public Circle(int x, int y, int radius, DrawAPI drawAPI) {
      super(drawAPI);
      this.x = x;  
      this.y = y;  
      this.radius = radius;
   }
 
   public void draw() {
      drawAPI.drawCircle(radius,x,y);
   }
}
//5、使用 Shape 和 DrawAPI 类画出不同颜色的圆。
public class BridgePatternDemo {
   public static void main(String[] args) {
      Shape redCircle = new Circle(100,100, 10, new RedCircle());
      Shape greenCircle = new Circle(100,100, 10, new GreenCircle());
 
      redCircle.draw();
      greenCircle.draw();
   }
}
